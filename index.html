Cloud Security AWS Architecture

Cloud Security — AWS Network Security Hardening

Led AWS network security hardening — designing phased NACL controls and tightening Security Group port restrictions across the cloud environment.

Problem

AWS environment relied on default NACLs with permissive rules and overly broad Security Group port configurations. Subnet-level segmentation gaps existed alongside unnecessarily open ports across workloads.

Approach

Designed a phased approach: tightened Security Group rules to restrict ports to only what each workload requires, implemented custom NACLs as subnet-level guardrails per workload group. Prepared architecture-specific questions for AWS TAM consultation. Coordinated with Network Team for implementation.

Outcome

Implemented dedicated custom NACLs per subnet/workload group and enforced least-privilege port access through Security Group hardening. Established defense-in-depth network segmentation across the AWS environment.
